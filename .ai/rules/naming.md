# Naming Rules — open-noc-ai

## Python

Use:

- snake_case for variables and functions
- PascalCase for classes
- UPPER_CASE for constants

## Models

Use singular names:

- Customer
- Site
- Carrier
- Link
- Device

## Endpoints

Use plural routes:

- /customers
- /sites
- /carriers
- /links
- /devices

## Schemas

Use clear schema names:

- EntityCreate
- EntityUpdate
- EntityRead
- EntityResponse

## CRUD Functions

Use clear crud function names:

- create_entity
- get_entity
- list_entities
- update_entity
- delete_entity

## Forbidden

Do not use:

- vague names
- abbreviations without meaning
- generic names like data, obj or item when domain name is clearer
